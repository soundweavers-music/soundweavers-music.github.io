# **第 03 堂：通用 DAW 架構：取樣率、位元深度與 Buffer 設定**

歡迎來到第三堂課！在前面兩堂課中，我們像是在打造一間實體的廚房：買了高級的鍋碗瓢盆（錄音介面與麥克風），也處理了廚房的通風與動線（房間的聲學處理與吸音）。現在，我們終於要正式踏入「數位」的世界，打開那套將陪伴你無數個熬夜創作夜晚的軟體——**DAW（數位音樂工作站，Digital Audio Workstation）**。

無論你是使用 Logic Pro、Pro Tools、Cubase、Studio One，還是免費強大的 Reaper，當你第一次興奮地打開軟體，點擊「新建專案（New Project）」時，通常不會馬上看到漂亮的混音台或音軌，而是會先跳出一個枯燥的設定視窗，上面寫著一些令人眼花撩亂的英文與數字：**Sample Rate (取樣率)、Bit Depth (位元深度)、Buffer Size (緩衝區大小)**。

很多初學者在這個階段會感到恐懼，隨便按個「OK」就開始錄音，直到專案變得龐大、電腦開始發出宛如直升機起飛的風扇狂轉聲，或是錄出來的聲音充滿了可怕的「波波波」雜音時，才發現地基從一開始就打歪了。

這堂課，我們不背艱澀的工程學定義。我們將用最生活化的比喻，帶你一次搞懂這三個 DAW 世界中最核心的「三大基石」。只要搞懂它們，你就能在任何一套軟體中游刃有餘，讓你的電腦發揮最大的效能，同時保持最完美的音質。

## **一、聲音是如何被「塞」進電腦裡的？（類比轉數位）**

在介紹那三個神秘參數之前，我們必須先弄懂一個非常基礎，但超級重要的概念：**電腦其實是個「聾子」**。

電腦聽不懂你對著麥克風唱的優美旋律，也聽不懂吉他琴弦震動的浪漫。在現實世界中，聲音是一種連續不斷的空氣震動，我們稱為「類比訊號（Analog Signal）」，它就像是一條平滑、連續且沒有斷點的優雅曲線。

但是，電腦的世界裡只有「0」和「1」。為了讓電腦理解聲音，錄音介面裡面有一個非常勤勞的員工，叫做 **A/D 轉換器（Analog to Digital Converter）**。這個員工的工作，就是拿著一台超級高速的相機，對著你唱出來的連續聲波「瘋狂拍照」。

他會記錄下每一個瞬間聲波的高度（音量），然後把這些無數張的照片轉換成 0 和 1 的數據，儲存在電腦的硬碟裡。當你按下播放鍵時，另一個員工 **D/A 轉換器** 就會把這些照片像放電影一樣快速翻閱，還原成連續的聲音播給你的耳機聽。

了解這個「瘋狂拍照」的過程後，我們就可以無縫接軌地理解接下來的兩個最重要設定：**取樣率（拍照的速度）** 與 **位元深度（照片的畫素/色彩豐富度）**。

## **二、取樣率 (Sample Rate)：聲音的「影格率」**

當你打開 DAW，第一個要設定的就是 Sample Rate（取樣率）。常見的選項有：44.1kHz, 48kHz, 96kHz, 甚至是 192kHz。

### **1\. 什麼是取樣率？**

用最白話的方式來說，取樣率就是**錄音介面一秒鐘對你的聲音拍幾張照片**。 「Hz（赫茲）」是頻率的單位，代表「每秒幾次」。而「k」是千的意思。所以，當你設定為 44.1kHz 時，代表錄音介面在一秒鐘之內，對著你唱的聲音拍了 **44,100 張照片**！

想像一下小時候玩的「手翻書（Flipbook）」。如果你一秒鐘只翻 5 頁，動畫看起來就會卡卡的、很不連貫；如果你一秒鐘翻 60 頁，動畫就會變得超級流暢，這就是影片世界裡的「影格率（FPS）」。在聲音的世界裡，取樣率就是聲音的影格率。一秒鐘拍 44,100 張照片，聽起來已經是一個天文數字了吧？為什麼需要拍這麼快呢？

### **2\. 為什麼是 44.1kHz？（奈奎斯特理論的白話文）**

人類的耳朵非常敏銳，我們能聽到的聲音頻率範圍，大約是 **20Hz（極低的低音）到 20,000Hz（極高的高音）**。 在數位音訊科學裡，有一條非常有名的鐵則叫做「奈奎斯特定理（Nyquist Theorem）」。這條定理告訴我們：**為了要完美記錄並還原某個頻率的聲音，你的取樣率必須要是那個頻率的「兩倍以上」。**

既然人類最高能聽到 20,000Hz，那麼我們只需要一秒鐘拍 40,000 張照片（40kHz），就能完美捕捉人類能聽到的所有聲音了。為了預留一點緩衝空間（給防雷射濾波器運作的空間），工程師們把標準定在了 **44,100Hz（44.1kHz）**。這就是為什麼幾十年來，CD 唱片的標準音質一直都是 44.1kHz 的原因。

### **3\. 我該選哪一個？48kHz 還是 192kHz？**

很多新手會有「數字越大、音質越好」的迷思，於是一買到錄音介面，就把 DAW 的取樣率狂拉到 192kHz。這其實是一場災難！

* **為什麼不建議用 192kHz？** 如果 44.1kHz 已經能完美涵蓋人類聽覺極限，那麼 192kHz 捕捉到的就是蝙蝠和海豚在聊天的聲音了（人類根本聽不到的高頻）。設定成 192kHz 不僅對聽感沒有明顯的幫助，還會帶來兩個致命的缺點： 第一，檔案變得超級無敵大，你的硬碟很快就會被塞爆。 第二，你的電腦 CPU 負載會直接暴增四倍。當你在混音時多掛幾個效果器，電腦可能就會直接當機崩潰。  
* **現代錄音的黃金標準：48kHz** 在現今的音樂與影音產業中，**48kHz 已經成為最通用的標準**。為什麼呢？因為 48kHz 是所有影片（YouTube, Netflix, 電影）的標準聲音格式。如果你未來做的音樂有可能會搭配影像發布在 YouTube 上，或是幫短影音配樂，直接使用 48kHz 可以避免影片剪輯軟體在轉換格式時產生音質損耗或影音不同步的問題。

**💡 宅錄生存法則一：** 打開 DAW 的第一步，無腦將專案 Sample Rate 設定為 **48kHz**。這是在「極佳音質」與「電腦效能」之間最完美的平衡點。

## **三、位元深度 (Bit Depth)：聲音的「色彩豐富度與動態範圍」**

設定好拍照速度後，下一個選項是 Bit Depth（位元深度），常見的選項有 16-bit, 24-bit, 甚至最新的 32-bit float。

### **1\. 什麼是位元深度？**

如果取樣率是一秒鐘拍幾張照片，那麼位元深度就是「每一張照片的畫素與色彩豐富度」**。 在聲音的世界裡，我們不看色彩，我們看的是**「音量（振幅）」的細膩度，專業術語叫做「動態範圍（Dynamic Range）」。

想像你手上有一把用來測量音量大小的尺。

* 如果你選 **16-bit**，這把尺上刻有 **65,536** 個刻度。這已經非常細膩了，這就是傳統 CD 唱片的標準。  
* 如果你選 **24-bit**，這把尺上會暴增到擁有 **16,777,216** 個刻度！

### **2\. 為什麼我們需要這麼多刻度？**

刻度越多，代表我們越能精準地捕捉到聲音從「極小聲的呼吸」到「極大聲的嘶吼」之間的細微變化，而且不會產生奇怪的數位階梯狀雜音。

更重要的是，位元深度決定了你的「底噪（Noise Floor）」有多低。 在 16-bit 的時代，如果你錄音的時候麥克風音量開得太小，事後在 DAW 裡面把音量拉大，你會同時把深藏在底部的「沙沙沙」數位底噪一起拉得超級大聲，整首歌會變得很髒。 但在 **24-bit** 的世界裡，因為刻度高達一千六百萬個，底噪被推到了極限深遠的深淵裡（理論上高達 144 分貝的動態範圍）。這意味著什麼呢？

這意味著「容錯率」！ 在 24-bit 的設定下，你錄音時「寧可錄小聲一點，也絕對不要錄到破音（Clipping）」。就算你錄進來的聲音波形看起來很小，事後在混音階段用壓縮器或推桿把它拉大，聲音依然會保持晶瑩剔透，完全不會有底噪暴增的問題。這給了我們極大的 Headroom（動態預留空間）。

### **3\. 32-bit Float（32 位元浮點）又是什麼魔法？**

近年來，很多高階錄音介面或外拍錄音機開始主打 32-bit float 功能。這是一個堪稱「物理外掛」的規格。 在傳統 24-bit 下，如果你的聲音太大聲，超出了 DAW 的 0 dBFS 極限，聲音頂端就會被「削平」，產生極度難聽的數位破音，這段錄音就毀了，只能重錄。 但在 32-bit float 的環境下，因為它使用了「浮點運算」的數學魔法，它的動態範圍大到超越了地球上任何能發出聲音的物體（高達 1500多分貝）。也就是說，**使用 32-bit float 錄音，聲音「絕對不會破音」**。就算波形在螢幕上看起來已經紅爆成一塊實心磚，你只要把音量拉下來，優美的聲音竟然奇蹟般地完好無缺！

不過，對於絕大多數的宅錄使用者，因為我們的麥克風和環境通常不會有像爆炸一樣無法預測的極端音量變化。

**💡 宅錄生存法則二：** 在建立專案時，將 Bit Depth 設定為 **24-bit**。這是業界錄音的絕對標準。如果你用的是 16-bit，你會綁手綁腳；如果你用 32-bit float，雖然不會破音，但檔案大小會增加約 50%。24-bit 給了你足夠的乾淨度與容錯空間，這點無庸置疑。

## **四、緩衝區大小 (Buffer Size)：延遲與效能的蹺蹺板**

接下來這個設定，是所有初學者最容易卡關，也是每天在錄音與混音切換時，必須不斷手動調整的參數：**Buffer Size（緩衝區大小）**。

通常這個設定不會出現在「新建專案」的畫面上，而是藏在 DAW 的「偏好設定 (Preferences) \-\> 音訊 (Audio)」裡面。你會看到一排數字可以選：32, 64, 128, 256, 512, 1024, 2048 (單位是 Samples)。

### **1\. 什麼是 Buffer？（咖啡廳服務生的比喻）**

想像你的電腦 CPU 是一間咖啡廳的廚師，錄音介面是客人，而 Buffer 就是負責端盤子的「服務生手上的托盤」。

當你對著麥克風唱歌，聲音資料會先放到服務生的托盤（Buffer）上，等托盤裝滿了，服務生才會端進廚房給 CPU 處理，處理完再端出來給你聽到。

* **如果托盤很小（例如 64 或 128 Samples）：** 服務生只要拿到一點點聲音資料，就會立刻衝進廚房再衝出來。 **優點：** 速度超快！你對著麥克風唱歌，耳機裡幾乎「同步」聽到自己的聲音，這叫做「低延遲（Low Latency）」。 **缺點：** 服務生（CPU）會跑到快氣喘發作！如果這時你的專案裡掛了很多耗資源的混音效果器，CPU 處理不及，聲音就會開始出現可怕的「啵啵啵、啪啪啪」破音，甚至 DAW 會直接停止播放並跳出「CPU Overload」的警告。  
* **如果托盤很大（例如 1024 或 2048 Samples）：** 服務生可以慢條斯理地等托盤裝了滿滿的聲音資料後，再走進廚房。 **優點：** 廚師與服務生（CPU）非常輕鬆愜意。你的電腦可以同時掛載上百個效果器、幾十軌龐大的虛擬管弦樂器，都不會卡頓。 **缺點：** 速度超級慢！等你唱完一句歌詞，可能過了半秒鐘，耳機裡才傳出你的聲音。這種「高延遲（High Latency）」會讓你大腦嚴重錯亂，完全無法對準節拍錄製任何樂器或人聲。

### **2\. 黃金切換法則：錄音與混音是兩種不同的狀態**

了解了這個蹺蹺板原理，你就會明白，世界上沒有一個「永遠完美」的 Buffer Size 設定，你必須根據你當下在做什麼事情，隨時切換它。

**【情境 A：我要開始錄音了！】** 當歌手站在麥克風前，或是你準備彈奏 MIDI 鍵盤時，你最需要的是「即時反饋」。 👉 **請把 Buffer Size 調小（設定在 64 或 128 Samples）。** 此時，因為要減輕 CPU 負擔，你應該把專案裡那些不必要、會吃資源的華麗效果器（特別是掛在 Master 總軌上的母帶效果器、或是極耗運算的 Reverb）暫時關閉。讓電腦專心處理錄音的低延遲傳輸。

**【情境 B：錄完了，我要開始混音了！】** 所有的音軌都錄好了，你不再需要對著麥克風發出聲音。你現在要做的是把這幾十條音軌加上 EQ、壓縮器、殘響，開始精雕細琢。 👉 **請把 Buffer Size 調大（設定在 1024 或 2048 Samples）。** 在混音階段，按下空白鍵播放後晚個零點幾秒才發出聲音，對你來說一點影響都沒有。但把托盤變大，你的電腦就能釋放出極大的運算能力，讓你隨心所欲地掛載各種夢幻的 Plug-ins（外掛效果器），而不會聽到煩人的爆音。

**💡 宅錄生存法則三：** **錄音時調小（128），混音時調大（1024）。** 把這句口訣印在腦海裡，甚至貼在你的螢幕旁邊。未來只要遇到錄音時耳機聲音慢半拍，或是混音時電腦一直爆音卡頓，第一件事就是去檢查你的 Buffer Size！

## **總結：你的第一個完美專案地基**

讓我們來複習一下這堂課的核心重點，當你下一次打開你的數位音樂工作站（DAW）時，請帶著自信，進行以下的三個神聖儀式：

1. **專案設定 Sample Rate（取樣率）：** 選擇 **48kHz**。確保你的音頻有著不妥協的高解析度，並與未來的影像世界無縫接軌。  
2. **專案設定 Bit Depth（位元深度）：** 選擇 **24-bit**。享受那深不見底的低底噪，並在錄音時給自己留足寬裕的 Headroom（動態空間），永遠不要錄到紅燈破音。  
3. **系統設定 Buffer Size（緩衝區大小）：** 要錄音就選 **128**，要混音就選 **1024**。靈活調配你電腦 CPU 的體力，成為掌控效能與延遲的大師。

這三個參數就像是數位音樂世界的地基。地基打穩了，不管你未來在專案裡蓋出多麼雄偉的音樂高樓，都不會輕易崩塌。你再也不會被電腦裡那些神秘的數字嚇到，而是能真正讓電腦成為你創作音樂的最佳助手。

準備好要發出聲音了嗎？在下一堂課中，我們將離開螢幕，回到現實世界。我將教你如何像個專業錄音師一樣，為你的麥克風找到最完美的位置。我們「第 04 堂：麥克風擺位與人聲錄製」見！
